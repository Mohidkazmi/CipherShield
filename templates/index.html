About Hybrid Shield

A Secure File Encryption System using Hybrid Cryptography

☯

Hybrid Cryptography

Combines RSA-2048 (asymmetric) for secure key exchange with AES-256-GCM (symmetric) for fast, authenticated file encryption. This is the same approach used by HTTPS/TLS and PGP.

✍

Digital Signatures

Uses RSA-PSS with SHA-256 to provide non-repudiation. A signer cannot deny having signed a file, and any modification to the signed file will invalidate the signature.

🔑

Key Management

A local Key Management System (KMS) to generate, track, and manage RSA key pairs. Keys are stored as industry-standard PEM files (PKCS#8 format).

⬢

Cryptographic Hashing

SHA-256 (256-bit, collision-resistant) for data integrity verification. MD5 (128-bit, broken) included for educational comparison only.

⬡

Symmetric Encryption

AES-256 via Fernet with PBKDF2HMAC key derivation (50,000 iterations + random salt). Converts weak human passwords into strong cryptographic keys.

🛡️

Authenticated Encryption

AES-GCM mode provides both confidentiality and integrity. A 128-bit authentication tag ensures that encrypted files cannot be tampered with undetected.
